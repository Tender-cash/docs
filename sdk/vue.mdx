## Overview

The Tender Vue SDK provides a simple component library for integrating the Tender Cash payment flow into your Vue 3 application. The SDK uses Shadow DOM for style isolation and provides a seamless payment experience.

## Usage

Pass all payment parameters directly as props. The modal will automatically open when the component mounts with the required payment parameters.

```vue
<script setup>
import { TenderAgentSdk } from '@tender-cash/agent-sdk-vue';

const handleEventResponse = (response) => {
  console.log('SDK Response:', response);
  // Handle success, partial payment, overpayment, error based on response.status
};
</script>

<template>
  <TenderAgentSdk
    access-id="YOUR_ACCESS_ID"
    fiat-currency="USD"
    env="test"
    @event-response="handleEventResponse"
    :amount="150.00"
    reference-id="unique-payment-reference-123"
    :payment-expiry-seconds="1800"
  />
</template>
```

<Info>
  The modal opens automatically on component mount when `reference-id` and `amount` are provided as props.
</Info>

## API Reference

### Component Props

The `TenderAgentSdk` component accepts the following props:

#### Required Props

| Prop            | Type                                | Description                                                                                                                                                 |
| --------------- | ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `fiat-currency` | `String`                            | The fiat currency code (e.g., "USD", "EUR").                                                                                                                |
| `access-id`     | `String`                            | Your Tender Cash merchant Access ID.                                                                                                                        |
| `env`           | `"test"` \| `"live"` \| `"local"`  | The environment to use (`"test"` for testing, `"live"` for production, `"local"` for local development).                                                   |

#### Optional Props

| Prop                     | Type                | Description                                                                       |
| ------------------------ | ------------------- | --------------------------------------------------------------------------------- |
| `reference-id`           | `String`            | Payment reference ID. Required to auto-open the modal.                            |
| `amount`                 | `Number`            | Payment amount in fiat currency. Required to auto-open the modal.                 |
| `payment-expiry-seconds` | `Number`            | Payment expiry time in seconds. Optional, defaults to 30 minutes.                 |
| `theme`                  | `"light"` \| `"dark"` | Theme for the payment modal. Optional, defaults to "light".                     |

#### Events

| Event            | Payload Type         | Description                                                  |
| ---------------- | -------------------- | ------------------------------------------------------------ |
| `event-response` | `OnFinishResponse`   | Emitted on payment completion or status change.              |

<Note>
  When `reference-id` and `amount` are provided as props, the modal will automatically open on component mount.
</Note>

## Using Template Ref to Control Modal

You can use a template ref to programmatically control the modal from outside the widget:

```vue
<script setup>
import { ref } from 'vue';
import { TenderAgentSdk } from '@tender-cash/agent-sdk-vue';

const tenderRef = ref(null);

const handleOpenPayment = () => {
  tenderRef.value?.initiatePayment({
    amount: 150.00,
    referenceId: "unique-payment-reference-123",
    paymentExpirySeconds: 1800
  });
};

const handleCloseModal = () => {
  // Close the modal programmatically from outside the widget
  tenderRef.value?.closeModal();
};

const handleEventResponse = (response) => {
  console.log('SDK Response:', response);
};
</script>

<template>
  <div>
    <button @click="handleOpenPayment">Open Payment</button>
    <button @click="handleCloseModal">Close Modal</button>
    
    <TenderAgentSdk
      ref="tenderRef"
      access-id="YOUR_ACCESS_ID"
      fiat-currency="USD"
      env="test"
      @event-response="handleEventResponse"
      :amount="150.00"
      reference-id="unique-payment-reference-123"
      :payment-expiry-seconds="1800"
    />
  </div>
</template>
```

### Ref Methods

| Method            | Description                                                            |
| ----------------- | ---------------------------------------------------------------------- |
| `initiatePayment` | Opens the modal and initiates a payment with the provided parameters.  |
| `dismiss`         | Closes the modal.                                                      |
| `closeModal`      | Closes the modal - can be called from outside the widget.              |

## Event Response Data

The `event-response` event emits an object with the following structure:

```typescript
interface OnFinishResponse {
  status: "partial-payment" | "completed" | "overpayment" | "pending" | "error" | "cancelled";
  message: string;
  data: IPaymentData | undefined; // Contains details like amountPaid, coin, address, etc.
}

interface IPaymentData {
  id?: string;
  amount?: number;
  coinAmount?: number;
  coin?: string;
  chain?: string;
  address?: string;
  amountPaid?: string;
  balance?: string;
  status?: "partial-payment" | "completed" | "overpayment" | "pending" | "error" | "cancelled";
}
```

### Handling Payment Status

```vue
<script setup>
const handleEventResponse = (response) => {
  switch (response.status) {
    case 'completed':
      console.log('Payment completed successfully!');
      console.log('Transaction ID:', response.data?.id);
      break;
    case 'partial-payment':
      console.log('Partial payment received');
      console.log('Amount paid:', response.data?.amountPaid);
      console.log('Balance:', response.data?.balance);
      break;
    case 'overpayment':
      console.log('Overpayment received');
      break;
    case 'pending':
      console.log('Payment is pending confirmation');
      break;
    case 'error':
      console.error('Payment error:', response.message);
      break;
    case 'cancelled':
      console.log('Payment was cancelled');
      break;
  }
};
</script>
```

## Complete Example (Composition API)

Here's a complete example showing payment initiation and status handling:

```vue
<script setup>
import { ref } from 'vue';
import { TenderAgentSdk } from '@tender-cash/agent-sdk-vue';

const tenderRef = ref(null);
const paymentStatus = ref('');

const handlePayment = () => {
  tenderRef.value?.initiatePayment({
    amount: 150.00,
    referenceId: `order-${Date.now()}`,
    paymentExpirySeconds: 1800 // 30 minutes
  });
};

const handleEventResponse = (response) => {
  paymentStatus.value = response.status;
  
  if (response.status === 'completed') {
    // Payment successful - update order status, show confirmation, etc.
    console.log('Payment successful!', response.data);
  } else if (response.status === 'partial-payment') {
    // Handle partial payment
    console.log('Partial payment received:', response.data?.amountPaid);
  } else if (response.status === 'error') {
    // Handle error
    console.error('Payment failed:', response.message);
  }
};
</script>

<template>
  <div>
    <h1>Checkout</h1>
    <button @click="handlePayment">Pay $150.00</button>
    
    <p v-if="paymentStatus">
      Payment Status: {{ paymentStatus }}
    </p>
    
    <TenderAgentSdk
      ref="tenderRef"
      :access-id="import.meta.env.VITE_TENDER_ACCESS_ID"
      fiat-currency="USD"
      env="test"
      @event-response="handleEventResponse"
      theme="light"
    />
  </div>
</template>
```

## Best Practices

<AccordionGroup>
  <Accordion title="Secure Credentials" icon="lock">
    - Store Access ID in environment variables (`.env` files)
    - Use `import.meta.env.VITE_TENDER_ACCESS_ID` for Vite projects
    - Never commit credentials to version control
    - Use different credentials for test and production
  </Accordion>
  
  <Accordion title="Reference IDs" icon="fingerprint">
    - Use unique reference IDs for each payment
    - Include order/transaction identifiers
    - Store reference IDs in your database for reconciliation
  </Accordion>
  
  <Accordion title="Event Handling" icon="bell">
    - Always implement `@event-response` handler
    - Handle all possible status values
    - Log errors for debugging and monitoring
  </Accordion>
  
  <Accordion title="Payment Expiry" icon="clock">
    - Set reasonable expiry times (default: 30 minutes)
    - Inform users about expiry duration
    - Handle expired payments gracefully
  </Accordion>
  
  <Accordion title="Reactive State" icon="arrows-rotate">
    - Use Vue's reactivity system for payment status
    - Update UI based on payment status changes
    - Clean up refs on component unmount
  </Accordion>
</AccordionGroup>

```typescript
const tenderRef = ref<InstanceType<typeof TenderAgentSdk> | null>(null);

  console.log('Status:', response.status);

};
```
